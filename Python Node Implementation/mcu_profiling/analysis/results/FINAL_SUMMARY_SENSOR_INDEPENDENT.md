## 📡 3. Mode 6 (Transmission - 전송)

**센서 무관 - Byte 크기에 따라 다름**

### Layer별 전송 (실제 측정값)

| Layer | 출력 크기 | 전송 시간 (ms) | 전송 전력 (mW) | 총 에너지 (mJ) | Byte당 에너지 (µJ) |
|-------|---------|---------------|---------------|---------------|-------------------|
| Layer 0 | **400 bytes** | 30.50 | 838.25 | 24,555.83 | 61,390 |
| Layer 1 | **320 bytes** | **138.00** ⚠️ | 731.17 | 70,284.69 | 219,640 |
| Layer 2 | **240 bytes** | **137.61** ⚠️ | 825.40 | 69,073.34 | 287,806 |
| Layer 3 | **160 bytes** | 26.75 | 887.37 | 23,596.74 | 147,480 |
| Layer 4 | **80 bytes** | 27.80 | 806.26 | 22,226.50 | 277,831 |

> **⚠️ 중요 발견**: 
> - Layer 1, 2의 전송 시간이 다른 Layer보다 **4-5배 더 김!**
> - 이는 데이터 크기와 무관하게 특정 Layer에서 전송 지연 발생
> - ESP-NOW 프로토콜 특성이거나 구현 이슈일 가능성

### 전송 에너지 특징
1. **전송 전력**: 평균 731-887 mW (매우 높음!)
2. **Byte당 에너지**: 데이터 크기에 비례하지 않음
   - 작은 데이터일수록 오버헤드 비중이 큼
3. **Layer 1, 2 최적화 필수**: 전송 시간이 비정상적으로 김

## 💡 최적화 권장 사항

### 1. 🎯 센서 선택
✅ **accel+gyro 조합 사용** (439.48 mW)
- 약 15% 에너지 절감 (vs. accel+gyro+mag)

### 2. ⚡ 전송 최적화 **(가장 중요!)**
⚠️ 전송이 가장 높은 전력 소비 (평균 731-887 mW)

**권장 사항:**
- **Layer 1, 2 전송 최적화 필수!** (138ms는 비정상적으로 김)
- Layer별로 매번 전송하지 말고 **배치 전송** 고려
- 데이터 **압축** 활용
- 작은 데이터(Layer 4)는 Byte당 에너지가 매우 높으므로 여러 결과를 모아서 전송

### 3. 🔧 레이어 효율
✅ Layer 4가 MFLOP당 에너지가 가장 낮음 (2,613.45 mJ/MFLOP)
- 가능하면 초기 레이어 경량화
- ESP32-C3는 하드웨어 FPU가 약하여 연산 효율이 낮음 (일반 AI 가속기보다 수천 배 낮음)

### 4. 💤 대기 전력 관리
⚠️ Idle 상태도 434.85 mW 소비
- **Deep sleep 모드** 적극 활용 권장
- 필요 시에만 활성화

## 📈 평균 소비 전력 순위

**전력 소비 낮음 → 높음:**

1. **Idle**: 434.85 mW (대기)

3. **Layer 추론**: 460-499 mW (거의 비슷)
4. **Transmission**: 731-887 mW ⚠️ **(최고! 약 2배 높음)**

## 📊 주요 발견사항

### ✅ 좋은 점
1. Layer들의 에너지 효율이 거의 비슷함 (2,600-2,800 mJ/MFLOP)
2. Layer 4가 가장 효율적 (2,613.45 mJ/MFLOP)
3. accel+gyro 센서 조합이 가장 효율적 (439.48 mW)

### ⚠️ 개선 필요
1. **전송 전력이 너무 높음** (731-887 mW) - 최우선 최적화 대상
2. **Layer 1, 2 전송 지연 이슈** (138ms) - ESP-NOW 구현 검토 필요
3. Byte당 에너지가 데이터 크기에 비례하지 않음 (프로토콜 오버헤드)
4. Idle 상태도 상당한 전력 소비 (434.85 mW)
5. ESP32-C3의 낮은 연산 효율 (하드웨어 FPU 부족)

### 💡 핵심 결론
**전송 최적화 = 전체 에너지 절감의 핵심!**
- 전송 횟수를 5회 → 1회로 줄이면 약 185,000 mJ 절감 (전체의 3%)
- Layer 1, 2 전송 시간을 138ms → 30ms로 개선하면 약 120,000 mJ 절감
