## 🔋 실전 에너지 계산 예시

### 시나리오: 전체 추론 1회 실행 (Layer 0~4 + 각 Layer 후 전송)

**에너지 구성:**
1. Layer 0 실행: 614 J
2. Layer 0 전송 (400 bytes): 16-22 J
3. Layer 1 실행: 792 J
4. Layer 1 전송 (320 bytes): 16-231 J
5. Layer 2 실행: 1,188 J
6. Layer 2 전송 (240 bytes): 16-84 J
7. Layer 3 실행: 1,596 J
8. Layer 3 전송 (160 bytes): 16-84 J
9. Layer 4 실행: 1,911 J
10. Layer 4 전송 (80 bytes): 16-27 J

**총 에너지 (추론 + 전송):**
- 최소: ~6,165 J (추론) + ~80 J (전송) = **~6,245 J**
- 최대: ~6,165 J (추론) + ~448 J (전송) = **~6,613 J**

**총 소요 시간:**
- 추론: 1,305 + 1,603 + 2,400 + 3,207 + 3,845 = **12,360 ms (약 12.4초)**
- 전송: 평균 ~150 ms
- **합계: 약 12.5초**

**센싱 포함 시 (3초):**
- 센싱: 439-506 mW × 3000 ms = 1,318-1,518 J
- **전체: 7,563-8,131 J (약 15.5초)**

## 💡 최적화 권장 사항

### 1. 센서 선택
✅ **accel+gyro 조합** 사용 (439.48 mW)
- 약 15% 에너지 절감 (vs. accel+gyro+mag)

### 2. 전송 최적화
⚠️ 전송이 가장 높은 전력 소비 (600-970 mW)
- 배치 전송 고려
- 데이터 압축 활용
- Layer별 전송 횟수 최소화

### 3. 레이어 효율
✅ Layer 4가 MFLOP당 에너지가 가장 낮음 (2,613 J/MFLOP)
- 가능하면 초기 레이어 경량화

### 4. 대기 전력 관리
⚠️ Idle 상태도 415-459 mW 소비
- Deep sleep 모드 적극 활용 권장
- 필요 시에만 활성화

## 📈 에너지 소비 순위

**모드별 전력 소비 (낮음 → 높음):**
1. Idle: ~435 mW (대기)
2. Sensing: 439-506 mW (센서 동작)
3. Layer 추론: ~470-498 mW (연산)
4. **Transmission: 600-970 mW (전송) ← 최고!**

**결론:** 전송 최적화가 전체 에너지 절감의 핵심!
